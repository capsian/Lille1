1.1 Exercice 1 question 3

Pour chaque dé, les probabilités de faire un jet de valeur paire ou impaire sont égales \[Pr(\text{valeur paire}) = Pr(\text{valeur impaire}) = \frac{1}{2}\]

Lorsque l'on jette les deux dés, il y a 2 situations dans lesquelles le résultat est paire :

Parité premier dé Parité du second dé
impaire impaire
paire paire

Comme ces situations sont disjointes et que les dés prennent des valeurs inépendantes l'une de l'autre nous avons : \[ \begin{array}{rcl} Pr(\text{somme paire}) &=& Pr(\text{premier dé impair} \cap \text{deuxième dé impaire}) + Pr(\text{premier dé paire} \cap \text{deuxième dé paire})\\ &=& Pr(\text{premier dé impaire}) \times Pr(\text{second dé impaire}) + Pr(\text{premier dé paire}) \times Pr(\text{second dé paire})\\ &=& \frac{1}{2}\times \frac{1}{2} + \frac{1}{2}\times\frac{1}{2}\\ &=& \frac{1}{2} \end{array} \]

1.2 Exercice 2

On prend comme espace probabiliste les paires de booléens. Une paire (m,p) signifiant qu'un patient est malade ssi m=True et que son test est positif ssi p=True.

Les données de l'énoncé sont :

  • Pr(test positif | malade) = 0.999
  • Pr(test Positif | non malade) = 0.005
  • Pr(malade) = 0.02

On cherche à déterminer Pr(malade | test positif).

Par définition de la probabilité conditionnelle, cela revient à chercher :

\[Pr(\text{malade} \mid \text{test positif}) = \frac{Pr(\text{malade} \cap \text{test positif})}{Pr(\text{test positif})}\]

Pour obtenir notre résultat, il nous faut déterminer les deux probabilités suivantes :

  1. Pr(malade ∩ test positif)
  2. Pr(test positif)

Calculons d'abord Pr(malade ∩ test positif). Par définition, nous avons : \[Pr(\text{test positif} \mid \text{malade}) = \frac{Pr(\text{malade} \cap \text{test positif})}{Pr(\text{malade})}\] Ainsi \[\begin{array}{rcl} Pr(\text{malade}\cap \text{test positif}) &=& Pr(\text{test positif} \mid \text{malade}) Pr(\text{malade})\\ &=&0.999\times 0.02=0.01998 \end{array}\]

Calculons maintenant Pr(test positif). Pour cela on remarque que : \[Pr(\text{test positif}) = Pr(\text{test positif} \cap \text{malade}) + Pr(\text{test positif} \cap \text{non malade})\]

Comme plus haut, nous avons : \[\begin{array}{rcl} Pr(\text{test positif} \cap \text{non malade}) &=& Pr(\text{test positif} \mid \text{non malade}) Pr(\text{non malade})\\ &=& Pr(\text{test positif} \mid \text{non malade}) (1-Pr(\text{malade}))\\ &=& 0.005 (1-0.02) = 0.0049 \end{array}\]

Pour conclure : \[Pr(\text{malade} \mid \text{test positif}) = 0.01998/(0.0049+ 0.01998) =0.8\]

Il s'ensuit qu'une personne est malade lorsque son test est positif avec une probabilité de 80%.

1.3.3 Question 3

La probabilité qu'un tour de boucle en déclenche un autre est r = pp + (1-p)(1-p). C'est lorsque t1 est égal à t2.

L'algorithme effectue au moins n tours de boucles avec une probabilité rn.

La probilité que l'algorithme ne termine pas est \(\lim_{n \to \infty} r^n\).

Si p est différent de 0 et de 1, cette limite tend vers 0.

En effet, \(r = p^2 + 1-2p + p^2 = 2p^2 -2p +1 = 2p(p-1) +1\).

Lorsque 0<p<1, nous avons : -1<p-1 <0. Ainsi -2 < 2p(p-1) < 0 et donc -1 < 2p(p-1) + 1 < 1. Nous avons bien -1 < r <1 et la limite tend vers 0.

L'algorithme termine avec probabilité 1.

1.3.4 Question 4

On calcule la probabilité de renvoyer True de 2 manières suivantes :

  1. Méthode 1 : calcul complet

    Nous commençons par calculer la probabilité de renvoyer True après k tours de boucle. Cette probabilité est

    \[r^{k-1} p(1-p)\]

    Comme, lorsque k ≠ k', les évènements "renvoyer True après k tours de boucle" et "renvoyer True après k' tours de boucle" sont disjoints, la probabilité de renvoyer True est :

    \[\begin{array}{rcl} \sum_{k=1}^\infty r^{k-1} p(1-p) &=& p(1-p)\sum_{k=0}^\infty (2p(p-1)+1)^k\\ &=& p(1-p)\frac{1}{1-(2p(p-1)+1)}\\ &=& \frac{p(1-p)}{2p(1-p)}\\ &=& \frac{1}{2} \end{array} \]

    Rappel : on utilise l'identité suivante (somme des termes d'une suite géométrique de raison r t.q. 0 ≤ r < 1) : \[\sum_{k=0}^\infty r^k = \frac{1}{1-r}\]

  2. Méthode 2 : utilisation des probabilités conditionnelles

    1 = Pr(termine | True) = Pr(termine ∩ True)/Pr(True)

    Parce que renvoyer True c'est terminer.

    Pr(True) = Pr(termine ∩ True)

    Pr(True | termine) = Pr(termine ∩ True)/ Pr(termine) = Pr(temine ∩ True) = Pr(True)

    Or Pr(True | termine) = 1/2

    Puisque dans un tour de boucle qui termine les probabilités de renvoyer True ou False sont égales.

1.4 Exercice 4

1.4.1 Question 1

Avant la boucle, v = 1 et c = 0. L'ensemble {0,…,v-1} est dans ce cas réduit à {0}. Comme Pr(c = 0) = 1, l'invariant est bien vérifié avant la boucle.

En début de boucle, en supposant que v=p, l'invariant est vérifié et ainsi que pour toute valeur k dans {0,…,p-1}, Pr(c=k) = 1/p.

Après les deux affectations de début de boucle, v=2p et c prend valeur dans {0,…,2p-1}. De plus, pour k dans cet ensemble, on pose l le résultat de la division euclidienne de k par 2. Comme la probabilité que flip() prenne valeur 0 ou 1 est 1/2, quelque soit la parité de k, la probabilité que c ait pour valeur k est 1/2 fois la probabilité qu'en début de boucle c avait pour valeur l. Ainsi la Pr(c=k)= 1/2p = 1/v.

Lorsque v ≥ n, on suppose que v prend pour valeur p. Par l'invariant, pour chaque valeur k de {n,…, p-1}, Pr(c=k) = 1/v. Ainsi, Pr(c=k | c ≥ n) = 1/(p-n). Il s'ensuit que lorsque l'on affecte la valeur p-n dans v et c-n dans c, pour tout k dans {0,…,p-n-1} la probabilité que c prenne pour valeur k est la probabilité qu'auparavant c avait pour valeur k+n sachant que c≥n. Cette probabilité est 1/(p-n). Ainsi c prend une valeur dans {0,…,p-n-1} avec une probabilité uniforme.

1.4.2 Questions 2 et 3

La valeur contenue dans v est strictement inférieure à 2n. En effet, l'algorithme termine forcément si v prend valeur n. Car alors c prend une valeur dans {0,…,n-1} et alors l'algorithme s'arrête.

Pour que l'algorithme renvoie une valeur il faut que v contienne une valeur p supérieure ou égale à n. Comme vue précédemment, p est dans ce cas strictement inférieure à 2n. Ainsi la probabilité que l'algorithme utilise un tour de boucle supplémentaire est 1 - n/p. Or cette probabilité est inférieure à 1-1/2= 1/2 (car p<2n). Ainsi, lorsque v contient une valeur supérieure à n, l'algorithme s'arrête avec une probabilité d'au moins 1/2. La probabilité que l'algorithme ne s'arrête pas est bornée par la limite \(\lim_{n\to\infty} \left(\frac{1}{2}\right)^n\) qui vaut 0. L'algorithme termine avec probabilité 1.

1.4.3 Question 4

Lorsque v a pour valeur p ≥ n, l'algorithme termine si c contient une valeur k dans {0,…,n-1}. Il nous faut évaluer la probabilité suivante : Pr(c=k | k<n). Par définition :

Pr(c=k | k<n) = Pr(c=k ∩ k<n)/Pr(k<n) = (1/p)/(n/p) = 1/n

Ainsi l'algorithme renvoie une valeur dans {0,…,n-1} avec une probabilité uniforme.
